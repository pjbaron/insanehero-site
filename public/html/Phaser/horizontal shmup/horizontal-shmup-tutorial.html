Preface

Title: "Crafting a Horizontal Scrolling Shooter in Phaser"

Welcome to this tutorial on creating a horizontal scrolling shooter, commonly known as a "horizontal shmup" (shoot 'em up), using Phaser!

A horizontal shmup is a subgenre of shoot 'em up games where the player's character, typically a spaceship or aircraft, moves from left to right across the screen. The gameplay involves dodging obstacles and enemy fire while shooting at incoming enemies and collecting power-ups. Classic examples include R-Type, Gradius, and Thunder Force.

In this tutorial, we'll focus on the core elements that make a horizontal shmup:

1. Creating a continuously scrolling background
2. Implementing player movement and shooting mechanics
3. Spawning and managing enemies
4. Handling collisions between player bullets and enemies

By the end of this tutorial, you'll have a solid foundation for building your own horizontal scrolling shooter game using Phaser. Let's dive in and start creating an exciting, action-packed experience!

src/main.js

Our main.js file is the central hub of our game. It imports all the necessary scenes and sets up the game configuration.

import { Boot } from './scenes/Boot.js';
import { Game } from './scenes/Game.js';
import { GameOver } from './scenes/GameOver.js';
import { MainMenu } from './scenes/MainMenu.js';
import { Preloader } from './scenes/Preloader.js';

const config = {
    type: Phaser.AUTO,
    width: 800,
    height: 600,
    parent: 'game-container',
    backgroundColor: '#000000',
    scale: {
        mode: Phaser.Scale.FIT,
        autoCenter: Phaser.Scale.CENTER_BOTH
    },
    scene: [
        Boot,
        Preloader,
        MainMenu,
        Game,
        GameOver
    ],
    physics: {
        default: 'arcade',
        arcade: {
            gravity: { y: 0 },
            debug: false
        }
    }
};

export default new Phaser.Game(config);


We start by importing all our game scenes. These scenes represent different states or screens in our game, like the boot process, preloader, main menu, actual gameplay, and game over screen.

Next, we define our game configuration. This object tells Phaser how to set up our game. Let's break down some key properties:

- `type: Phaser.AUTO` tells Phaser to use WebGL if available, falling back to Canvas if not.
- `width` and `height` set the dimensions of our game.
- `parent` specifies which HTML element will contain our game.
- `backgroundColor` sets the default background color.
- The `scale` property ensures our game fits nicely in different screen sizes.
- `scene` is an array of all the scenes in our game, in the order they'll be initialized.
- The `physics` property sets up Phaser's physics system. We're using the arcade physics system with no gravity, which is perfect for our space-themed shooter.

Finally, we create a new Phaser.Game instance with our configuration and export it. This is what kicks off our entire game.

src/scenes/Boot.js

The Boot scene is the first to run when our game starts. Its main job is to load any assets needed for the preloader scene.

export class Boot extends Phaser.Scene
{
    constructor ()

        super('Boot');
    }

    }

    create ()
    {
        this.scene.start('Preloader');
    }
}


In the `preload` method, we load our background image. This image will be used in the preloader scene to give the player something to look at while the rest of the assets load.

Once the background is loaded, the `create` method immediately starts the Preloader scene. Think of this Boot scene as a quick preparation step before the main asset loading begins.

src/scenes/Preloader.js

The Preloader scene is responsible for loading all the main assets our game will need. It also displays a loading bar to give the player feedback on the loading progress.

export class Preloader extends Phaser.Scene
{
    constructor ()
    {
        super('Preloader');
    }

    preload ()
    {
        this.add.image(400, 300, 'background');

        const progressBar = this.add.graphics();
        const progressBox = this.add.graphics();
        progressBox.fillStyle(0x222222, 0.8);
        progressBox.fillRect(240, 270, 320, 50);

        const width = this.cameras.main.width;
        const height = this.cameras.main.height;
        const loadingText = this.make.text({
            x: width / 2,
            y: height / 2 - 50,
            text: 'Loading...',
            style: {
                font: '20px monospace',
                fill: '#ffffff'
            }
        });
        loadingText.setOrigin(0.5, 0.5);

        this.load.on('progress', (value) => {
            progressBar.clear();
            progressBar.fillStyle(0xffffff, 1);
            progressBar.fillRect(250, 280, 300 * value, 30);
        });

        this.load.on('complete', () => {
            progressBar.destroy();
            progressBox.destroy();
            loadingText.destroy();
        });

        // Load assets here
        this.load.image('player', 'assets/player.png');
        this.load.image('enemy', 'assets/enemy.png');
        this.load.image('bullet', 'assets/bullet.png');
    }

    create ()
    {
        this.scene.start('MainMenu');
    }
}


In the `preload` method, we first display the background image we loaded in the Boot scene. Then we create a progress bar and loading text to show the player that assets are being loaded.

We use Phaser's event system to update the progress bar as assets load. The 'progress' event fires repeatedly during loading, allowing us to update the bar's width. The 'complete' event fires when all assets are loaded, at which point we clean up our loading graphics.

At the end of the `preload` method, we load our game assets: the player ship, enemy ship, and bullet images.

Once all assets are loaded, the `create` method starts the MainMenu scene.

1. Scrolling Background

In the Game scene's create method, add the following code:

create() {
    this.bg1 = this.add.tileSprite(0, 0, 800, 600, 'background').setOrigin(0, 0);
    this.bg2 = this.add.tileSprite(800, 0, 800, 600, 'background').setOrigin(0, 0);
}


This creates two tiled background sprites that we'll use to create a scrolling effect. We position the second sprite right after the first one.

In the update method, add:

update() {
    this.bg1.tilePositionX += 2;
    this.bg2.tilePositionX += 2;

    if (this.bg1.x <= -800) {
        this.bg1.x = 800;
    }
    if (this.bg2.x <= -800) {
        this.bg2.x = 800;
    }
}


This code moves both background sprites to the left by 2 pixels each frame, creating a scrolling effect. When a sprite moves completely off-screen to the left, we reposition it to the right side of the screen.

2. Player Ship

Add the following to the Game scene's create method:

this.player = this.physics.add.sprite(100, 300, 'player');
this.player.setCollideWorldBounds(true);


This creates a player sprite with physics enabled, positioned at x=100 and y=300. The `setCollideWorldBounds(true)` call prevents the player from moving off-screen.

3. Player Movement

In the create method, add:

this.cursors = this.input.keyboard.createCursorKeys();


This sets up keyboard input. Then, in the update method:

if (this.cursors.up.isDown) {
    this.player.setVelocityY(-200);
} else if (this.cursors.down.isDown) {
    this.player.setVelocityY(200);
} else {
    this.player.setVelocityY(0);
}


This allows the player to move up and down using the arrow keys. The velocity of 200 (or -200) pixels per second gives smooth movement.

These additions provide a scrolling background and a movable player ship, setting the stage for the next steps in our game development process.

1. src/scenes/Game.js
- Add collision detection between player bullets and enemies
- Implement the score system
- Create a game over condition when enemies reach the left side
- Add the enemy spawning system

// Create bullet group
this.bullets = this.physics.add.group({
    classType: Bullet,
    maxSize: 10,
    runChildUpdate: true
});

// Create enemy group
this.enemies = this.physics.add.group({
    classType: Enemy,
    runChildUpdate: true
});

// Set up collision between bullets and enemies
this.physics.add.collider(this.bullets, this.enemies, this.bulletHitEnemy, null, this);

// Set up collision between player and enemies
this.physics.add.collider(this.player, this.enemies, this.gameOver, null, this);

// Set up score
this.score = 0;
this.scoreText = this.add.text(16, 16, 'Score: 0', { fontSize: '32px', fill: '#fff' });

// Enemy spawning
this.time.addEvent({
    delay: 2000,
    callback: this.spawnEnemy,
    callbackScope: this,
    loop: true
});

// Check for game over
this.enemies.getChildren().forEach((enemy) => {
    if (enemy.x <= 0) {
        this.gameOver();
    }
});

spawnEnemy ()
{
    const y = Phaser.Math.Between(50, 550);
    const enemy = this.enemies.get();
    if (enemy) {
        enemy.spawn(800, y);
    }
}

bulletHitEnemy (bullet, enemy)
{
    bullet.destroy();
    enemy.destroy();
    this.score += 10;
    this.scoreText.setText('Score: ' + this.score);
}

gameOver ()
{
    this.registry.set('score', this.score);
    this.scene.start('GameOver');
}


2. src/gameObjects/Bullet.js (New file)

export class Bullet extends Phaser.Physics.Arcade.Image
{
    constructor(scene, x, y)
    {
        super(scene, x, y, 'bullet');
    }

    fire(x, y)
    {
        this.setPosition(x + 50, y);
        this.setActive(true);
        this.setVisible(true);
        this.setVelocityX(300);
    }

    preUpdate(time, delta)
    {
        if (this.x > 800)
        {
            this.setActive(false);
            this.setVisible(false);
        }
    }
}


3. src/gameObjects/Enemy.js (New file)
- Create an Enemy class extending Phaser.GameObjects.Sprite
- Implement enemy movement and destruction when off-screen or hit by a bullet

export class Enemy extends Phaser.Physics.Arcade.Sprite
{
    constructor(scene, x, y)
    {
        super(scene, x, y, 'enemy');
    }

    spawn(x, y)
    {
        this.setPosition(x, y);
        this.setActive(true);
        this.setVisible(true);
        this.setVelocityX(-100);
    }

    preUpdate(time, delta)
    {
        if (this.x < -50)
        {
            this.setActive(false);
            this.setVisible(false);
        }
    }
}
